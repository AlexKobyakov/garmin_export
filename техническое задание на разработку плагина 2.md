# 📑 Техническое задание

## на разработку плагина QGIS для экспорта карт в формат Garmin IMG через mkgmap

---

### 1. Общая информация

**Наименование проекта:**
Плагин QGIS «Garmin Exporter» (рабочее название).

**Цель разработки:**
Создание плагина QGIS, позволяющего напрямую экспортировать векторные данные (Shape, GeoPackage, PostGIS и др.) в формат Garmin IMG через библиотеку **mkgmap** с возможностью управления стилями, уровнями масштабов и пакетным экспортом всех слоёв проекта.

**Аналоги:**

* mkgmap (консольная утилита).
* GPSMapEdit (ручное создание MP → IMG).

**Отличие:**
Прямое встраивание в QGIS с графическим интерфейсом, автоматическим мэппингом стилей и многослойной компиляцией.

---

### 2. Функциональные требования

#### 2.1. Основные функции

1. Экспорт данных QGIS в формат `.mp`.
2. Компиляция `.mp` → `.img` через **mkgmap**.
3. Возможность указания выходного каталога и имени файла.
4. Поддержка системных форматов QGIS: Shapefile, GeoPackage, PostGIS, CSV с координатами.
5. Экспорт сразу всех слоёв проекта (`QgsProject.instance().mapLayers()`).

#### 2.2. Сопоставление стилей QGIS → Garmin TYP

1. Используется JSON-мэппинг.
2. Мэппинг сопоставляет геометрию, атрибуты и стили QGIS с Garmin-типами (`0xNN`).
3. Плагин подгружает JSON автоматически и применяет к слоям.
4. Возможность ручного редактирования JSON через GUI плагина.

**Пример структуры JSON:**

```json
{
  "layers": {
    "roads": {
      "geometry": "LineString",
      "type": "0x06",
      "label_field": "name",
      "style": {
        "color": "#FF0000",
        "width": 2
      },
      "level": 1
    },
    "rivers": {
      "geometry": "LineString",
      "type": "0x1F",
      "label_field": "river_name",
      "style": {
        "color": "#0000FF",
        "width": 1
      },
      "level": 2
    },
    "forests": {
      "geometry": "Polygon",
      "type": "0x16",
      "label_field": "forest_name",
      "style": {
        "fill_color": "#00FF00"
      },
      "level": 3
    }
  }
}
```

#### 2.3. Масштабирование (уровни карт Level0...Level3)

1. Поддержка до 4 уровней (`Level0...Level3`).
2. В `.mp` файле создаются секции `LevelX` и `ZoomX`.
3. В JSON-мэппинге у каждого слоя можно указать `level`.
4. Если не задано — используется дефолтный `Level1`.

#### 2.4. Управление слоями проекта

1. Плагин получает все слои через `QgsProject.instance().mapLayers()`.
2. В GUI отображается список всех слоёв с галочками.
3. Пользователь может:

   * Экспортировать все слои.
   * Экспортировать выбранные слои.
   * Настраивать параметры через JSON.

#### 2.5. GUI (графический интерфейс)

1. Дизайн интерфейса **должен быть идентичен плагину**
   *MIF/TAB to SHP/GeoJSON Converter* (папка `C:\AlexKo\mif_to_shp_converter_3.6`).
2. Основные элементы:

   * Список слоёв (с галочками).
   * Выбор JSON-файла мэппинга.
   * Настройки уровней масштабов.
   * Путь к `mkgmap.jar`.
   * Кнопка «Скомпилировать».

#### 2.6. Локализация

1. Поддержка локализаций (многоязычность).
2. Система перевода **идентична реализации в плагине**
   *MIF/TAB to SHP/GeoJSON Converter* (папка `C:\AlexKo\mif_to_shp_converter_3.6`).
3. Языки по умолчанию: EN, RU.

### 3. Нефункциональные требования

#### 3.1. Архитектура и модульность

1. Реализация плагина должна быть максимально модульной.
2. Каждый компонент системного файла плагина — **не более 700 строк кода**.
3. Архитектура должна повторять принципы плагина *MIF/TAB to SHP/GeoJSON Converter*.

#### 3.2. Совместимость

1. QGIS ≥ 3.22.
2. Python 3.9+.
3. Поддержка Windows, Linux, macOS.

#### 3.3. Производительность

1. Экспорт до 500 тыс. объектов должен выполняться ≤ 60 сек (на ПК среднего уровня).
2. Возможность многопоточной компиляции при использовании mkgmap.
