# 📝 Техническое задание

на разработку плагина для QGIS: **«Экспорт проекта в Garmin IMG через mkgmap»**

---

## 1. Общие сведения

**Наименование:** Плагин QGIS «Garmin Export»
**Назначение:** Автоматизация экспорта векторных данных (слоёв QGIS) в формат карт Garmin `.IMG`, с использованием внешнего компилятора `mkgmap`.
**Используемая среда:** QGIS (≥3.16), Python 3 (PyQGIS), Java (для `mkgmap`).

---

## 2. Цель разработки

Создать плагин для QGIS, позволяющий пользователю:

* выбрать один или несколько слоёв проекта (например, SHP, GeoPackage, GeoJSON);
* сопоставить их со схемой типов объектов Garmin;
* автоматически преобразовать данные в формат `.mp` (Polish format);
* запустить `mkgmap` для компиляции в `.img`;
* сохранить готовый `.img`-файл для загрузки в устройство Garmin.

---

## 3. Функциональные требования

### 3.1. Интерфейс

* В меню QGIS появляется новый пункт: **Плагины → Garmin Export**.
* Открывается диалоговое окно (PyQt), содержащее:

  1. Список доступных слоёв проекта (с галочками выбора).
  2. Поле выбора каталога для сохранения `.img`.
  3. Поле выбора `mkgmap.jar` (путь к Java-компилятору).
  4. Кнопку «Сопоставление стилей» (открывает окно с JSON-таблицей mapping).
  5. Параметры сборки:

     * Family ID (FID)
     * Map ID
     * Name (название карты)
     * Transparent (да/нет)
     * Routing (да/нет)
  6. Кнопку **«Скомпилировать»**.

### 3.2. Обработка данных

* Каждый выбранный слой преобразуется в формат `.mp`:

  * **Точечные слои → POI (Point of Interest)**
  * **Линейные слои → POLYLINE (дороги, реки, границы)**
  * **Полигональные слои → POLYGON (леса, водоёмы, районы)**
* Атрибуты (например, `name`) используются как `Label` в `.mp`.
* Координаты берутся в WGS84 (EPSG:4326).

### 3.3. Генерация `.mp`

* Для каждого слоя формируется секция `.mp` вида:

```plaintext
[POLYLINE]
Type=0x06
Label=Main Road
Data0=(55.7558,37.6173),(55.7568,37.6193)
End
```

* Допускается использование уровней `Level0...Level3` для разных масштабов.
* Схема преобразования хранится в JSON (пример: `"road": "0x06"`).

### 3.4. Компиляция через `mkgmap`

* После генерации `.mp` вызывается команда:

```bash
java -jar mkgmap.jar --gmapsupp --family-id=1234 --description="QGIS Map" output.mp
```

* Результат: `gmapsupp.img` или `map.img` в указанной папке.

### 3.5. Логирование

* В окне плагина отображается прогресс и сообщения `mkgmap`.
* Ошибки записываются в лог-файл плагина (`~/.qgis3/garmin_export.log`).

Отлично 🙌 Давай добавим эти функции прямо в **ТЗ** и распишем структуру JSON-мэппинга.


### 3.6. Сопоставление стилей QGIS → Garmin TYP

* Для сопоставления объектов между QGIS и Garmin будет использоваться **mapping-файл в формате JSON**.
* В этом файле определяются правила преобразования:

  * Название слоя или его атрибут → Garmin Type (в формате `0xNN`).
  * Дополнительно можно указывать: цвет линии, ширину, подпись.
* Пользователь может редактировать JSON-файл через интерфейс плагина («Сопоставление стилей»).

### 📄 Структура JSON-мэппинга

```json
{
  "layers": {
    "roads": {
      "geometry": "LineString",
      "type": "0x06",
      "label_field": "name",
      "style": {
        "color": "#FF0000",
        "width": 2
      }
    },
    "rivers": {
      "geometry": "LineString",
      "type": "0x1F",
      "label_field": "river_name",
      "style": {
        "color": "#0000FF",
        "width": 1
      }
    },
    "forests": {
      "geometry": "Polygon",
      "type": "0x16",
      "label_field": "forest_name",
      "style": {
        "fill_color": "#00FF00"
      }
    },
    "poi": {
      "geometry": "Point",
      "type": "0x2f00",
      "label_field": "name",
      "icon": "tree"
    }
  }
}
```

### 🗝️ Объяснение структуры

* `geometry` — тип геометрии слоя (Point, LineString, Polygon).
* `type` — Garmin-код (шестнадцатеричный).
* `label_field` — атрибут слоя, используемый для `Label`.
* `style` — базовые параметры (цвет, толщина линии, заливка).
* `icon` — (опционально) имя иконки для POI.

---

### 3.7. Масштабирование (уровни карт Level0...Level3)

* В `.mp` файле можно задавать **несколько уровней отображения** объектов:

```plaintext
[IMG ID]
Level0=24
Level1=22
Level2=20
Level3=18
Zoom0=0
Zoom1=1
Zoom2=2
Zoom3=3
```

* Каждый объект при экспорте получает параметр `Levels`.
* В JSON-мэппинге можно указать уровень отображения:

```json
"roads": {
  "geometry": "LineString",
  "type": "0x06",
  "label_field": "name",
  "level": 1
}
```

## 3.8. Экспорт всех слоёв проекта

* Плагин должен уметь обрабатывать **все слои проекта автоматически**.
* Для этого используется API:

```python
from qgis.core import QgsProject

layers = QgsProject.instance().mapLayers().values()
for layer in layers:
    # экспорт каждого слоя в MP
```

* В интерфейсе плагина пользователь может:

  * Отметить галочками нужные слои (или выбрать все сразу).
  * Указать индивидуальные параметры экспорта (по JSON).

## 5. Входные и выходные данные

* **Входные:** проект QGIS с векторными слоями (`.shp`, `.gpkg`, `.geojson`).
* **Выходные:**

  * `.mp` — промежуточный файл Polish format
  * `.img` — итоговая карта Garmin
  * (опционально) `.typ` — файл оформления

## 8. Пример сценария использования

1. Пользователь открывает проект QGIS с несколькими SHP (дороги, реки, точки).
2. Запускает плагин **Garmin Export**.
3. Отмечает галочками нужные слои, указывает имя карты и FID.
4. Настраивает Сопоставление стилей QGIS → Garmin TYP или автоматическое (по-умолчанию) сопоставление
5. Настраивает необходимое отображение подписей полей из атрибутивной таблицы исходных векторных данных
6. Жмёт «Скомпилировать».
7. Получает файл `gmapsupp.img`, копирует его на Garmin в папку `/Garmin/`.
8. Активирует карту на устройстве.

---

## 9. Риски и ограничения

* Не все стили QGIS можно автоматически перенести в Garmin TYP.
* Ограничения Garmin по размеру карт (макс. 4 ГБ).
* Необходима установленная Java для запуска `mkgmap`.
